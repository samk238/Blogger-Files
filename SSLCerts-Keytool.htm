   cacerts  -  is a truststore. A trust store is used to authenticate the servers/peers.

KeyStore  -  is used to store the private keys of the clients so that it can share it to the server when the server requests client authentication.

 

TrustStore - is used to store certificates from trusted Certificate authorities(CA) which are used to verify certificate presented by Server in SSL Connection.

   KeyStore - is used to store private key and own identity certificate which program should present to other parties (Server or client) to verify its identity.

 

Certificate/Certification authority (CA) is an entity that issues digital certificates

 

Public key certificate, also known as a digital or identity certificate, is an electronic document used to prove the ownership of a public key.

     - certificate includes information about the key,

                                          information about the identity of its owner (called the subject) and

                                          the digital signature of an entity that has verified the certificates contents (called the issuer)

 

A certificate that has corresponding private key is also know as a digital ID.

 

-Djavax.net.ssl.keyStore to specify path for keyStore

-Djavax.net.ssl.trustStore to specify path for trustStore in Java.

-Dweblogic.security.SSL.trustedCAKeyStore=/app/oracle/Middleware/wlserver_10.3/server/lib/cacerts - weblogic

 

One-way SSL authentication creates a truststore on the client and a keystore on the server.

 

 

 

Two-way SSL (client-side authentication) creates a truststore and a keystore on both the client and the server.

By default windows/browsers on servers have certs installed?

  Every device includes something called a root store.

  A root store is a collection of pre-downloaded root certificates (and their public keys) that live on the device itself.

  Generally, the device will use whatever root store is native to its OS, otherwise it might use a third-party root store via an app like a web browser.

    - https://support.apple.com/en-us/HT209501#trusted

    - https://wiki.mozilla.org/index.php?title=CA/IncludedCertificates&redirect=no

why intermidiate in between?

   - provides an added level of security as the CA does not need to issue certificates directly from the CA root certificate.

   - If primary Root CA is not in the browser, the Intermediate CA must be installed on the server acting as a chain link between the browser root and the server certificate.
