JMS Queue vs Topic:

one msg consumed by one client/system - one to many

represent Point-To-Point model - represent Public and Subscribe model

FIFO - LIFO

 

Distributed Queue/ Topic: Defines a set of queues/topics that are distributed on multiple JMS servers, but which are accessible as a single, logical queue/topic to JMS clients.

 

ConnectionFactory:

 encapsulates connection configuration information, and enables JMS applications to create a Connection.

 Supports concurrent use, #enabling multiple threads to access the object simultaneously.

 You can use the preconfigured default connection factories provided by WebLogic JMS, or you can configure one or more connection factories to create connections with predefined attributes that suit your application.

 

Foreign JMS Server: configure the Foreign JMS system(External/ThirdParty JMS system or Weblogic's JMS System residing on another weblogic instance) and communicate with the Foreign JMS System's objects
