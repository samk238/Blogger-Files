For Versions Prior 10.3 - bsu                            $MW_HOME/utils/bsu/bsu.sh -report

For weblogic 12c and above - Opatch            $MW_HOME/OPatch/opatch lspatches

 

If you are not aware of what is your MW_HOME and what is your WL_HOME.

Look for the weblogic.jar file in your filesystem

$ find /apps -name "weblogic.jar"

Sample OP: /apps/oracle-weblogic/wls12213/wlserver/server/lib/weblogic.jar

/apps/oracle-weblogic/wls12213/wlserver  ----   is  WL_HOME

/apps/oracle-weblogic/wls12213        ----  is  MW_HOME

 

BSU:

#Verfiy

./bsu.sh -view  -patch_download_dir=$MW_HOME/utils/bsu/cache_dir    -prod_dir=$WL_HOME  -status=applied   -verbose

 

#Apply:

cp -Rf /app/binaries/Patches/WLS/${PATCH_NUMBER}/*         $MW_HOME/utils/bsu/cache_dir

./bsu.sh -install  -patch_download_dir=$MW_HOME/utils/bsu/cache_dir   -patchlist='${PATCH_LIST}'  -prod_dir=$WL_HOME  -verbose -log=error.txt

 

#Rollback:

./bsu.sh -remove  -patch_download_dir=$MW_HOME/utils/bsu/cache_dir   -patchlist='${PATCH_LIST}'   -verbose

 

***************************************************************************

Opatch:

#Verfiy:

./opatch lsinventory            before_`hostname`_`date +%Y%m%d`   -and-   after_`hostname`_`date +%Y%m%d`

 

#Apply:

cd ${PATCH_NUMBER}

./opatch apply                                                      

       ####Optional:

       Oracle Configuration Manager (OCM)

       ${ORACLE_HOME}/OPatch/ocm/bin/emocmrsp -no_banner -output  /tmp/ocm.rsp

      Add email and password for oracle support, if needed provide them, if not just press enter and response will be saved in the .rsp file, which can be reffered in future patches…

      ./opatch apply    -silent   -ocmrf  /tmp/ocm.rsp

 

#Rollback:

cd ${PATCH_NUMBER}

./opatch rollback -id ${PATCH_NUMBER} -silent

###################

Interim Patch:   A patch containing one or more fixes made available to customers who cannot wait until the next patch set or new product release to get a fix.

                   aka:  PSE, MLR, Exception release, One-Off, x-fix, Hotfix, Security One-Off

 

###################

Bundle Patch:  An iterative, cumulative patch that is issued between patch sets. Bundle patches usually include only fixes, but some products may include minor enhancements.

                    Ex: Database Windows Bundles and SOA Bundle Patches.

                  aka:   Maintenance Pack, Service Pack, MLRs, Cumulative Patch, Update Release

 

################

Security Patch Update (SPU):   An iterative, cumulative patch consisting of security fixes. Formerly known as Critical Patch Update.

                                          Note: The program name which delivers SPUs will still be called Critical Patch Update, as defined below:

             Critical Patch Update: Oracle's program for quarterly release of security fixes.

                                                     Patches released as part of this program may be Patch Set Updates, Security Patch Updates, and Bundle Patches.

                                                     Regardless of the patch type, the patches are cumulative.

                                            aka: Critical Patch Updates, CPUs

 

#################

Patch Set Updates (PSU):    Patch Set Updates are used to patch Oracle WebLogic Server only.

released on a quarterly basis, following the same schedule as the Critical Patch Updates (CPUs).

These come out the closest Tuesday to the 15th of the months of January, April, July and October.

Is intended to address the top fifty critical bugs affecting the broad customer base.
