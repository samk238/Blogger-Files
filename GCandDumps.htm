OOO - native (internal operations) or heap memory (Application related) - JDK   --    jcmd <pid> GC.run - run force GC

     Objects are allocated (eg new()) in the “heap” of the memory

     When new allocations for objects cannot happen in heap, we run into “java.lang.OutOfMemory”

 

Static members, class definitions, meta data, etc are stored in Permgen space

 

Thread dump(JDK): diagnose CPU spikes, deadlocks, poor response times, memory problems, unresponsive applications, and other system problems

   \\ Analyzing tools Samurai or TDA, JVisualVM, Java Mission Control (JMC),

 

1. When you send a SIGQUIT signal to the java process, a thread dump is sent to the stdout

        kill -3 PID

2. Using “weblogic.Admin”:

         java weblogic.Admin -url t3://AdminHostName:7001 -username weblogic -password weblogic THREAD_DUMP

         java weblogic.Admin -url t3://MS1HostName:7003 -username weblogic -password weblogic THREAD_DUMP

3.  Admin Console:   Server  –> Monitoring  –> Threads  ->  Click on the Dump Thread Stack

4. ${JAVA_HOME}/bin/jstack -l   >   threadDump.txt

 

JRockit:  jrcmd PID print_threads

 

Stack Dump  Force stack dump  -  $jstack -F PID   >   PID_stckdump.log   -  Stack memory is used to store local variables and function call

                                                                                                                                 - Xss, no memory left in the stack - java.lang.StackOverFlowError

 

Heap dump:  Eclipse memory analyzing tool         //Heap memory is used to store objects in Java/ Java objects are created in Heap

 

1.For Hotspot/Oracle JDK, you need to add the following parameters to JAVA_OPTIONS

  -XX:+HeapDumpOnOutOfMemoryError

  -XX:HeapDumpPath=<path>

2.Jmap, jmap -heap PID

  $JAVA_HOME/bin/jmap -dump:format=b,file=${FILE_NAME}.hprof  <PID>        

  $JAVA_HOME/bin/jmap -J-d64 -dump:format=b,file=${FILE_NAME}.hprof <PID>          Note: use -J-d64 if JVM is 64 Bit Jvm “jmap -J-d64 -heap pid”

 

Garbage collection is a mechanism provided by JVM to reclaim heap space from objects which are eligible

 

JRockit:

-jrcmd <PID> hprofdump filename=name_of_dump_file
