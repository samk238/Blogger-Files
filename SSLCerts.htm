SL Certificate bundle file (your_domain_name.p7b)

$keytool -import -trustcacerts -alias server -file your_domain_name.p7b -keystore your_site_name.jks

 

PEM (Privacy Enhanced Mail) format is the preferred format for private keys, digital certificates, and trusted certificate authorities (CAs)

Primary Certificate > Intermediate Certificate > Root Certificate. Save the file with a .pem extension

 

1. List the detailed information about installed certificates

   $ keytool -list -v -keystore cacert.jks

  

2. Generate CSR #Certificate Signing Request (CSR)

   $ keytool -certreq -v -alias server -file dnb.csr -keypass changeit123 -keystore cacert.jks -storepass changeit123

    #$keytool -genkey -alias server -keyalg RSA -keysize 2048 -keystore your_site_name.jks

 

3. Send the generated CSR to the Certificate Vendor to get the new certificate

     -----BEGIN NEW CERTIFICATE REQUEST----- and -----END NEW CERTIFICATE REQUEST----- tags

 

4. Verify the received cert:

   $ keytool -printcert -v -file cert.cer

  

5. Importing or installing the received certificate in keystore file

   $ keytool -import -v -alias  xxsomethingxx  -file  cert.cer  -keypass changeit  -keystore cacert.jks  -storepass  changeit123

 

6. Place the keystore file in the Weblogic/WebSphere domain SSL configuration directory and restart the whole domain.

 

6. Once installed the certificate verify the keystore file

   $ keytool -list -v -keystore cacert.jks

   Note: Observer the validation period and CN name

 

7. Place the keystore file in the Weblogic/WebSphere domain SSL configuration directory and restart the whole domain.

  How to generate new keystore file

  ****************************************

  $ keytool -genkey -v -alias qa-dnb.ges.symantec.com -keysize 1024 -dname "CN=qa-dnb.ges.symantec.com, OU=IT Security, O=Symantec Corporation, L=Mountain View, ST=California, C=US" -keyalg rsa -validity 365 -keypass changeit -keystore cacert.jks -storepass changeit123

 

  If there is a problem in executing above command to generate new keystore file then execute the below one and supply the values of CN Name, Organization, Organization Unit, Locality, State, Country.

  $ keytool -genkey -alias qa-dnb.ges.symantec.com -validity 730 -keystore cacerts.jks
