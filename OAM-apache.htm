Module mod_log_config - LogFormat and CustomLog directives

       The access log contains information about requests coming in to the web server

       The error log contains information about errors the web server encountered when processing requests, such as missing files.

 

400 -> Bad Request error   , HTTP request that was sent to the server has invalid syntax.

401 -> Unauthorized error  , user trying to access the resource has not been authenticated or has not been authenticated correctly.

403 -> Forbidden error     , valid request but the server is refusing to serve the request, due to a lack of permission to access the requested resource

404 -> Not Found error     , user is able to communicate with the server but it is unable to locate the requested file or resource.

500 -> Internal Server     , server cannot process the request for an unknown reason.

502 -> Bad Gateway error   , server is a gateway or proxy server, not receiving a valid response from the backend servers that should actually fulfill the request.

503 -> Service Unavailable , server is overloaded or under maintenance. This error implies that the service should become available at some point.

504 -> Gateway Timeout     , server is a gateway or proxy server, and it is not receiving a response from the backend servers within the allowed time period.
